LEARN TO CODE

Your first program is 30 seconds away.

Ten tiny lessons. Every example is editable and runs the real Spider compiler in your browser — change things, break things, and read what the compiler tells you: it's a teacher, not a judge.

1. Say something

say prints a line. Text lives between quotes. Press Run — congratulations, you're a programmer.

2. Name your values

let names a value that never changes; var names one that can. Weave any value into text with braces: {like_this}.

3. Ask questions

ask shows a question and hands back the answer as Text. (In the browser, type answers in the small box — one per line.)

4. Make decisions

if runs its indented lines only when the condition is true. Blocks are shown by indenting 4 spaces — what you see is what the computer sees.

5. Repeat yourself (without repeating yourself)

Three loops, each reading like a sentence: repeat 3 times, for item in items, and for i in 1 to 10.

6. Teach the computer a recipe

fn creates a function — a named recipe. Give it ingredients (parameters, with their types) and it can hand back a result with return.

7. Collections: lists and maps

A List keeps things in order; a Map connects keys to values. Positions count from 0.

8. Your own kinds of values

record bundles named fields; choice is exactly one of several cases; match handles every case — and Spider checks you didn't forget one. Try deleting the Dot line of the match and see.

9. When things can fail

Spider has no exceptions and no null. A function that can fail says so — it returns an Outcome — and try unwraps it with a fallback. Honest failure, no surprises.

10. Break something — on purpose

The most important lesson: errors are how the compiler teaches you. Run this as-is and read the message — what happened, why, how to fix. Then fix it and run again.

That's the whole core. Now spin something of your own.
